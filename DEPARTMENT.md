## 三、工作流程

### 3.1 任务处理流程

```
接收任务（从CEO/Product）
    ↓
理解需求（有问题及时沟通）
    ↓
技术方案设计（复杂任务）
    ↓
代码开发
    ↓
单元测试/自测
    ↓
Code Review
    ↓
提交测试
```

### 3.2 代码开发流程

```
1. 分支管理
   - 从develop创建feature分支
   - 分支命名：feature/功能名
   
2. 开发
   - 编写代码
   - 编写单元测试
   - 遵守编码规范
   
3. 自测
   - 功能测试
   - 边界测试
   - 异常测试
   
4. Code Review
   - 提交PR
   - 等待Review
   - 修复问题
   - 合并到develop
```

### 3.3 故障处理流程

```
发现故障
    ↓
初步定位（前端/后端/数据库）
    ↓
紧急修复
    ↓
测试验证
    ↓
发布修复
    ↓
故障复盘
```

## 五、代码规范

### 5.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 变量 | 小驼峰 | userName |
| 常量 | 全大写下划线 | MAX_COUNT |
| 函数 | 小驼峰 | getUser() |
| 类名 | 大驼峰 | UserService |
| 文件名 | 下划线 | user_service.dart |

### 5.2 Git提交规范

```
格式：type: description

类型：
- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- style: 格式调整
- refactor: 重构
- test: 测试相关
- chore: 构建/工具

示例：
- feat: 添加用户登录功能
- fix: 修复登录页面白屏问题
- docs: 更新API文档
```
