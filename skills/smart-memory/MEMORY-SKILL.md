# 🧠 智能记忆系统

> **作用**：管理 Agent 的记忆读取、写入和整理
> **适用**：所有 Agent（CEO / Developer / Product / Tester）
> **版本**：v2.1
> **更新日期**：2026-04-01

---

## 一、核心原则

```
❌ 过去的问题：
- 问"今天干了什么" → 不知道，靠猜
- 记忆不准确、不完整

✅ 现在的目标：
- 问"今天干了什么" → 读取 memory/今日.md → 准确回答
- 用户说了什么、Agent回复了什么、Agent干了什么 全部记录
```

## 三、启动读取流程

> ⚠️ **每次 session 启动时必须执行**

### 第一步：动态日期计算

```python
当前日期 = "2026-04-01"
今日文件名 = f"memory/{当前日期}.md"  # memory/2026-04-01.md
昨日日期 = 当前日期 - 1天  # 2026-03-31
昨日文件名 = f"memory/{昨日日期}.md"  # memory/2026-03-31.md
```

### 第二步：初始化今日记忆文件

```python
今日文件 = f"{工作空间}/memory/{今日日期}.md"

if not exists(今日文件):
    创建空文件(今日文件)
    写入文件头(今日文件, 今日日期)
else:
    直接读取
```

**文件头格式**：
```markdown
# YYYY-MM-DD 工作记录

> 创建时间: YYYY-MM-DD HH:MM
```

### 第三步：读取以下文件（按顺序）

```
1. IDENTITY.md      → 身份定位
2. USER.md          → 用户信息
3. MEMORY.md        → 长期记忆（项目、决策、团队）
4. memory/昨日.md   → 昨天干了什么
5. memory/今日.md   → 今天干了什么
6. AGENTS.md        → 团队成员和调度命令
7. TOOLS.md         → 可用工具配置
8. RULES.md         → 团队规范
```

**文件读取规则**：
```
for 文件 in 文件列表:
    if exists(文件):
        try:
            读取文件内容
        except 读取错误:
            记录错误，继续下一个
            print(f"⚠️ {文件} 读取失败，跳过")
    else:
        print(f"⏭️ {文件} 不存在，跳过")
        continue
```

### 第四步：Mem0 向量搜索

```python
# 根据不同Agent类型，使用不同的查询词
queries = {
    "CEO": [
        "当前项目进展和任务状态",
        "用户偏好和习惯", 
        "最近的重要决策",
        "团队协作问题"
    ],
    "Developer": [
        "项目开发进度",
        "技术问题和解决方案",
        "代码规范偏好",
        "GitHub/Supabase配置"
    ],
    "Product": [
        "PRD和需求分析历史",
        "用户偏好和产品方向",
        "竞品分析结论"
    ],
    "Tester": [
        "测试进度和BUG状态",
        "测试用例和结果"
    ]
}

for query in queries[当前Agent类型]:
    results = memory_search(query=query, scope="long-term", limit=3)
    整合到上下文
```

**Mem0 结果处理**：
```
如果找到相关记忆：
    → 将结果整合到当前上下文
    → 用于补充今日工作的背景信息

如果未找到：
    → 继续，不影响主流程
```

### 第五步：读取失败处理

```python
读取结果 = {
    "成功": [],
    "失败": [],
    "跳过": []
}

for 文件 in 文件列表:
    try:
        内容 = 读取文件(文件)
        读取结果["成功"].append(文件)
    except FileNotFoundError:
        读取结果["跳过"].append(文件)
    except Exception as e:
        读取结果["失败"].append((文件, str(e)))
```

## 四、智能写入流程

> ⚠️ **对话过程中实时执行，边对话边写入**

### 触发时机

```
┌─────────────────────────────────────────────────────────┐
│                    对话进行中                            │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  用户发送消息 ──→ 分析内容 ──→ 判断写入 ──→ 执行写入     │
│       ↑                              ↓                   │
│       └──────── 继续对话 ←────── 写入完成                 │
│                                                          │
└─────────────────────────────────────────────────────────┘

触发点：
1. 用户发送消息后 → 分析并写入
2. Agent 回复前 → 检查是否需要写入
3. Agent 执行操作后 → 记录操作
4. 每次关键操作完成 → 立即写入
```

### 写入来源

| 来源 | 说明 | 记录位置 |
|------|------|----------|
| **用户说了什么** | 需求、偏好、决策、反馈 | USER.md / MEMORY.md / memory/今日 |
| **Agent 回复了什么** | 分析、方案、确认、汇报 | memory/今日 |
| **Agent 干了什么** | 执行的操作、派发的任务、修改的文件 | memory/今日 |

#### 1. 用户偏好（优先级：最高）


```python
    if 主语 in ["我", "我们"]:
        写入目标 = "USER.md"
```

#### 3. 重要决策（优先级：高）

### 主语识别规则

```python
def 识别主语(文本):
    # 用户说的（第一人称）
    if 匹配(r"^我", 文本):
        return "用户"
    
    # 用户说的（第二人称）
    if 匹配(r"^你", 文本):
        return "Agent"
    
    # Agent说的
    if 匹配(r"^✅|^💾|^📖|^🔍", 文本):
        return "Agent"
    
    # 无主语，根据上下文判断
    return "上下文推断"
```

```python
# 记录Agent执行的操作
```

### 内存缓冲机制

```python
# 使用内存缓冲，减少磁盘IO

写入缓冲 = {
    "memory/今日.md": [],
    "USER.md": [],
    "MEMORY.md": []
}

def 添加到缓冲(目标文件, 内容):
    写入缓冲[目标文件].append({
        "时间": 当前时间(),
        "内容": 内容
    })

def 刷新缓冲():
    """将缓冲写入磁盘"""
    for 目标文件, 记录列表 in 写入缓冲.items():
        if 记录列表:
            for 记录 in 记录列表:
                追加写入文件(目标文件, 记录)
            清空(记录列表)
```

### 写入触发条件

```python
# 每隔一段时间或满足条件时，刷新缓冲

触发条件 = {
    "时间触发": "距离上次写入超过5分钟",
    "数量触发": "缓冲中超过10条记录",
    "关键操作触发": "完成重要操作（任务完成/决策确认）",
    "对话结束触发": "用户说'今天就这样'/'结束'"
}

if 满足任一触发条件:
    刷新缓冲()
```

### 去重检查

```python
def 需要写入(目标文件, 新内容):
    """检查是否需要写入"""
    
    # 读取目标文件
    if not exists(目标文件):
        return True  # 文件不存在，需要创建
    
    现有内容 = 读取文件(目标文件)
    
    # 完全相同检查
    if 新内容 in 现有内容:
        print(f"⏭️ 内容已存在，跳过: {新内容[:50]}...")
        return False
    
    # 相似度检查（关键词匹配）
    新内容关键词 = 提取关键词(新内容)
    for 行 in 现有内容.split("\n"):
        行关键词 = 提取关键词(行)
        相似度 = 计算相似度(新内容关键词, 行关键词)
        if 相似度 > 0.8:
            print(f"⏭️ 内容相似度{相似度*100:.0f}%，跳过: {新内容[:50]}...")
            return False
    
    return True  # 新内容，需要写入
```

## 五、每日结束汇总

> **每次对话结束时执行**

### 汇总触发条件

| 触发 | 说明 | 操作 |
|------|------|------|
| 用户说"今天就这样" | 用户主动结束 | 立即汇总 |
| 用户说"结束" | 用户主动结束 | 立即汇总 |
| 用户说"拜拜" | 用户主动结束 | 立即汇总 |
| 每天固定时间 18:00 | 定时触发 | 自动汇总 |
| 新的一天开始 | 日期变更 | 汇总昨天的 |

### 汇总执行步骤

```python
def 执行每日汇总():
    # 1. 读取今日记忆
    今日内容 = 读取文件(f"memory/{今日日期}.md")
    
    # 2. 提取关键信息
    完成的任务 = 提取_任务完成(今日内容)
    重要决策 = 提取_重要决策(今日内容)
    用户反馈 = 提取_用户反馈(今日内容)
    问题解决 = 提取_问题解决(今日内容)
    
    # 3. 生成汇总
    汇总 = 生成汇总报告(今日日期, 完成的任务, 重要决策, 用户反馈, 问题解决)
    
    # 4. 追加到今日记忆
    追加写入文件(f"memory/{今日日期}.md", 汇总)
    
    # 5. 打印汇总
    打印(汇总)
```

## 六、每周整理流程

> **每周五自动执行**

### 整理执行步骤

```python
def 执行每周整理():
    # 1. 收集本周所有memory文件
    本周文件 = []
    for 日期 in 本周日期列表:
        文件 = f"memory/{日期}.md"

    用户偏好 = 提取_用户偏好(本周文件)
    项目决策 = 提取_项目决策(本周文件)

    if 项目决策:
        更新_MEMORY.md(项目决策)
    
    # 4. 生成周总结
    周总结 = 生成周总结文件(本周文件)
    
    # 5. 保存周总结
    写入文件(f"memory/{周数}-周总结.md", 周总结)
    
    # 6. 清理（可选）
    # 删除一个月前的日文件
```

## 八、常见问题处理

### Q1: 读取的文件不存在？

```python
if not exists(文件路径):
    print(f"⏭️ {文件} 不存在，跳过")
    # 继续下一个文件，不影响整体流程
```

### Q2: 写入时文件被占用？

```python
try:
    写入文件(文件路径, 内容)
except FileExistsError:
    # 等待后重试
    time.sleep(0.5)
    写入文件(文件路径, 内容)
```

### Q3: Mem0查询没有结果？

```python
results = memory_search(query, limit=3)
if not results:
    print("🔍 Mem0: 未找到相关记忆")
    # 继续，不影响主流程
```

### Q4: 如何确保不遗漏？

```python
# 每次回复用户前执行
def 回复前检查():
    检查缓冲()
    if 缓冲数量 > 0:
        刷新缓冲()
    检查今日文件最后更新时间()
    if 距离上次更新 > 5分钟:
        提醒用户("记忆已更新")
```

## 九、记忆质量标准

### 准确性
```
□ 内容与实际对话一致
□ 时间戳正确
□ 不夸大、不遗漏
```

### 完整性
```
□ 用户输入完整记录

```

```
□ 格式统一
□ 分类清晰
□ 关键词突出
```
