# observability Skill

> **作用**：提供项目进度和 Agent 状态的实时可视化
> **参考**：OpenAI "Chrome DevTools Protocol" 可观测性实践
> **版本**：v1.0

---

## 一、核心概念

**可观测性 = 可见 + 可追溯 + 可诊断**

```
┌─────────────────────────────────────────────────────────┐
│                    可观测性三要素                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 度量（Metrics）    → 数值指标：进度%、Token消耗     │
│  2. 日志（Logs）       → 操作记录：谁做了什么            │
│  3. 追踪（Traces）    → 调用链：任务如何流转            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 四、指标定义

### 4.1 项目指标

| 指标 | 计算方式 | 目标 |
|------|---------|------|
| **完成度** | passed / total | 100% |
| **Bug密度** | P0+P1 bug数 / 代码行数 | < 1/1000 |
| **人均速度** | 功能数 / Agent数 / 天 | > 0.5 |
| **Token效率** | 功能数 / Token总消耗 | > 0.001 |

### 4.2 Agent 指标

| 指标 | 计算方式 | 告警阈值 |
|------|---------|---------|
| **Context 使用率** | tokens_used / context_window | > 40% |
| **Session 利用率** | 有效时间 / 总时间 | < 60% |
| **任务完成率** | 完成数 / 分配数 | < 80% |

### 4.3 质量指标

| 指标 | 计算方式 | 目标 |
|------|---------|------|
| **代码质量** | Evaluator 评分 | > 85 |
| **架构合规率** | 合规项 / 总项 | > 90% |
| **测试覆盖率** | 覆盖行 / 总行 | > 70% |
| **Lint 通过率** | 通过数 / 总数 | 100% |

## 五、日志规范

### 5.1 必需记录事件

```
[TIMESTAMP] [AGENT] [EVENT] [DETAILS]

示例：
2026-04-09T10:00:00+08:00 [Developer] [Session_Start] [Feature: F001]
2026-04-09T10:30:00+08:00 [Developer] [Commit] [Feature: F001, Commit: abc1234]
2026-04-09T11:00:00+08:00 [Tester] [Test_Start] [Feature: F001]
2026-04-09T11:30:00+08:00 [Tester] [Test_Complete] [Feature: F001, Result: Passed]
```

### 5.2 事件类型

| 事件 | 说明 |
|------|------|
| `Session_Start` | Agent Session 开始 |
| `Session_End` | Agent Session 结束 |
| `Commit` | 代码提交 |
| `Test_Start` | 测试开始 |
| `Test_Complete` | 测试完成 |
| `Bug_Found` | 发现 Bug |
| `Bug_Fixed` | Bug 修复 |
| `Quality_Check` | 质量检查 |
| `Context_Warning` | Context 告警 |
| `Handoff` | 任务交接 |

## 六、在工作流中的集成

### 6.1 CEO 每日检查

```
每日启动 CEO 时：
1. 运行 observability skill
2. 检查所有 Agent 状态
3. 查看是否有告警
4. 决定是否需要干预
```

### 6.2 Heartbeat 监控

```
每 30 分钟心跳：
1. 检查 Context 使用率
2. 检查 Token 消耗
3. 检查进度状态
4. 如有异常，发送告警
```
