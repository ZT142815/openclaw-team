# progress-tracking Skill

> **作用**：追踪长任务进度，支持断点续传
> **触发**：每次阶段完成时自动更新
> **版本**：v1.0

---

## 一、任务状态文件

每个进行中的项目都有一个 `progress.md`：

```markdown
# 项目进度追踪

## 项目信息
- 项目名称：[名称]
- 开始时间：[YYYY-MM-DD HH:mm]
- 当前阶段：[阶段]
- 预计完成：[预估时间]

## 阶段进度

### 1. 需求分析
- 状态：✅ 完成 / 🔄 进行中 / ⏳ 待开始
- 开始时间：[时间]
- 完成时间：[时间]
- 产出物：[文件]
- 备注：[说明]

### 2. PRD 撰写
- 状态：✅ 完成 / 🔄 进行中 / ⏳ 待开始
- 开始时间：[时间]
- 完成时间：[时间]
- 产出物：[文件]
- 备注：[说明]

### 3. 技术设计
- 状态：✅ 完成 / 🔄 进行中 / ⏳ 待开始
- ...

### 4. 开发
- 状态：✅ 完成 / 🔄 进行中 / ⏳ 待开始
- ...

### 5. 测试
- 状态：✅ 完成 / 🔄 进行中 / ⏳ 待开始
- ...

### 6. 交付
- 状态：✅ 完成 / 🔄 进行中 / ⏳ 待开始
- ...

## 整体进度

| 阶段 | 状态 | 开始 | 完成 | 产出物 |
|------|------|------|------|--------|
| 需求分析 | ✅ | 10:00 | 10:30 | - |
| PRD 撰写 | ✅ | 10:30 | 11:00 | PRD.md |
| 技术设计 | 🔄 | 11:00 | - | - |
| 开发 | ⏳ | - | - | - |
| 测试 | ⏳ | - | - | - |
| 交付 | ⏳ | - | - | - |

**总进度：33%**（2/6 阶段完成）

## 最近更新
- 11:00：PRD 撰写完成
- 11:00：开始技术设计

## 阻塞问题
- [无 / 问题描述]

## 下一步行动
1. 完成技术设计文档
2. 等待用户确认 PRD
```

---

## 二、阶段状态定义

| 状态 | 符号 | 说明 |
|------|------|------|
| 待开始 | ⏳ | 还未开始此阶段 |
| 进行中 | 🔄 | 正在执行 |
| 已完成 | ✅ | 阶段完成，等待下一阶段 |
| 阻塞 | 🚫 | 遇到问题无法继续 |

---

## 三、更新时机

```
自动更新进度：

1. CEO 派发任务 → 更新为"进行中"
2. Agent 输出 artifact → 记录产出物
3. Quality Gate 通过 → 更新为"已完成"
4. 遇到阻塞 → 记录问题
5. 用户确认节点 → 更新阶段
```

---

## 四、断点续传机制

### 保存检查点

```markdown
每次阶段完成时，自动保存检查点：

检查点内容：
├── 当前阶段
├── 已完成的产出物（文件路径）
├── 关键决策
├── 阻塞问题（如有）
└── 下一步行动
```

### 恢复流程

```
Session 断开
    ↓
新 Session 启动
    ↓
检测到进行中的任务
    ↓
读取 progress.md
    ↓
恢复上下文
    ↓
继续执行
```

## 六、与工作流的集成

### 流程中的更新点

```
1. CEO 分析需求
   → progress.md: "需求分析" = 🔄

2. CEO 派发 Product + Developer
   → progress.md: "PRD撰写" = 🔄, "技术设计" = 🔄

3. Product 输出 PRD
   → progress.md: 记录产出物路径

4. Quality Gate 1 通过（用户确认）
   → progress.md: "PRD撰写" = ✅, "技术设计" = ✅

5. Developer 开发
   → progress.md: "开发" = 🔄

6. Developer 自测通过
   → progress.md: 记录产出物

7. Tester 测试
   → progress.md: "测试" = 🔄

8. Quality Gate 2 通过
   → progress.md: "测试" = ✅

9. 交付完成
   → progress.md: "交付" = ✅, 总进度 = 100%
```

## 七、阻塞处理

```markdown
## 遇到阻塞时

1. 记录阻塞问题
   → progress.md: "阻塞问题" = "具体描述"

2. 通知 CEO
   → "⚠️ [项目] 在 [阶段] 遇到阻塞：
      问题：[描述]
      影响：[影响分析]
      建议：[解决方案]"

3. 等待决策
   → 用户决策后继续
```
